Problem statement and definitions#

Many applications of optimization require solutions that satisfy a set of constraints. For example, in a Vehicle Routing Problem (VRP), one seeks to route a set of trucks through a set of customer locations while minimizing the total travel time; constraints are needed to ensure that each customer is serviced by exactly one truck, that each truck starts its route at the depot and returns to it, and that truck capacities are not exceeded. In a workforce scheduling problem, workers are assigned tasks such that they perform at most one task at any given time and are not overworked overall.

Formally, a constraint should be seen as an expression of the form:

\[c'(\boldsymbol{x}) \leq b.\]

The left-hand side is the output of a real-valued function \(c': \mathbb{R}^n\mapsto\mathbb{R}\) that maps an assignment of values to the variables \(\boldsymbol{x}\) to a scalar value. That value is then compared to the right-hand side, \(b\in\mathbb{R}\). Constraints of the form \(c'(\boldsymbol{x}) \geq b\) can be expressed as \(-c(\boldsymbol{x}) \leq -b\) so we will use less-than-or-equal constraints without loss of generality. It is equivalent to redefine the left-hand side to \(c(\boldsymbol{x}) = c'(\boldsymbol{x}) - {b}\) so that an inequality constraint can be rewritten as:

\[c(\boldsymbol{x}) \leq 0.\]

For example, if a variable \(x_i\in\mathbb{R}\) represents the amount of money spent towards acquiring a certain product and there is a budget of \(b\) dollars, then \(c(\boldsymbol{x})=\sum_{i=1}^{n}{x_i}-b\). Bound constraints are also commonly used. For instance, you cannot spend a negative amount of money or more than a certain amount \(d\) for a specific product, meaning that \(c_{i1}(x_i)=x_i-d\), \(c_{i2}(x_i)=-x_i\) and the constraints are written as \(c_{i1}(x_i)\leq 0\) and \(c_{i2}(x_i)\leq 0\).

Alternatively, one may need to enforce an equality constraint:

\[c(\boldsymbol{x}) = 0.\]

We will use \([m]\) to denote the set of integers from \(1\) to \(m\) inclusive, i.e., \([m]=\{1,\cdots,m\}\). For a constrained optimization problem with \(m\) constraints, we will split the set \([m]\) of constraint indices into two disjoint subsets: \(\mathcal{I}\), the set of inequality constraints and \(\mathcal{E}\), the set of equality constraints. One of these sets may be empty and their union is equal to \(m\), i.e., \(\mathcal{I}\cup\mathcal{E}=[m]\). A continuous constrained optimization problem on \(n\) continuous variables can then be expressed as follows:

\[\begin{split}\min_{\boldsymbol{x}\in\mathbb{R}^n} \quad & f(\boldsymbol{x})\\ \text{subject to} \quad & c_i(\boldsymbol{x}) \leq 0, \quad\forall i\in\mathcal{I},\\ & c_i(\boldsymbol{x}) = 0, \quad\forall i\in\mathcal{E}.\end{split}\]

We will use \(\Omega=\{\boldsymbol{x}\in\mathbb{R}^n \;|\; c_i(\boldsymbol{{x}}) \leq 0, \forall i\in\mathcal{I}, c_i(\boldsymbol{{x}}) = 0, \forall i\in\mathcal{E}\}\) to denote the feasible set defined as the points of \(\mathbb{R}^n\) that satisfy all constraints. This allows us to rewrite the generic constrained optimization problem as:

\[\min_{\boldsymbol{x}\in\Omega} f(\boldsymbol{x}).\]

An minimization problem is unbounded if \(\Omega\) is non-empty but the constraints in \(\mathcal{I}\) and \(\mathcal{E}\) do not restrict the objective function \(f\), i.e., \(f(\boldsymbol{{x}})\) can be made arbitrarily small.

An optimization problem is infeasible if \(\Omega\) is empty, meaning that there does not exist any point in \(\mathbb{R}^n\) that simultaneously satisfies all constraints in \(\mathcal{I}\) and \(\mathcal{E}\).

With this new notation we can now define local and global minimizers of a constrained problem.

Definition 4 (Local minimizer of a constrained problem)

A weak local minimizer is a feasible point \(\hat{\boldsymbol{x}}\) that minimizes the objective function \(f\) within a neighborhood. Let \(\mathcal{N}=\{\boldsymbol{x}\in\mathbb{R}^n\;|\;\lVert \boldsymbol{x}-\hat{\boldsymbol{x}}\rVert<\delta\}\) be the neighborhood consisting of points within distance less than \(\delta\) from \(\hat{\boldsymbol{x}}\). Then, \(\hat{\boldsymbol{x}}\) is a local minimizer if there exists a \(\delta > 0\) such that \(f(\hat{\boldsymbol{x}}) \leq f(x)\) for all \({\boldsymbol{x}}\in\Omega\cap\mathcal{N}\).

A strong local minimizer is a weak local minimizer which is strictly better than all other points in a neighborhood, i.e., \(f(\hat{\boldsymbol{x}}) < f(x)\) for all \({\boldsymbol{x}}\in\Omega\cap\mathcal{N}, \boldsymbol{x} \neq \hat{\boldsymbol{x}}\).

The main difference between this definition and that of an unconstrained local minimizer from the previous chapter is that the neighborhood around \(\hat{\boldsymbol{x}}\) consists only of feasible points, i.e., points that are with \(\delta\) of \(\hat{\boldsymbol{x}}\) but that are infeasible in the constraint may have \(f(\hat{\boldsymbol{x}}) > f(x)\). Such points have better objective values but they are not of interest to us because they are infeasible.

Definition 5 (Global minimizer of a constrained problem)

A global minimizer is a feasible solution \(\boldsymbol{x^\star}\in\Omega\) that is as good as any other feasible solution, i.e., \(f({\boldsymbol{x^\star}}) \leq f(\boldsymbol{x})\) for all \(\boldsymbol{x}\in\Omega\). There may be multiple global minima.

Affine/linear constraints and halfspaces#

The dot product \(\boldsymbol{w}\cdot\boldsymbol{x}\) is the sum of the elementwise products of the two vectors, i.e., \(\boldsymbol{w}\cdot\boldsymbol{x}=\sum_{i=1}^{n}w_i x_i\).

Affine constraints are some of the most widely used in optimization. They can be expressed as \(c(\boldsymbol{x})\leq 0\) or \(c(\boldsymbol{x}) = 0\) where \(c(\boldsymbol{x})=\boldsymbol{w}\cdot\boldsymbol{x} - b\). The vector \(\boldsymbol{w}\in\mathbb{R}^n\) is the normal vector of the set of points that live on the affine hyperplane \(\boldsymbol{w}\cdot\boldsymbol{x} = b\). Note that an affine hyperplane is simply a plane for \(n=2\) and a line for \(n=1\).

When \(b=0\), we refer to \(\boldsymbol{w}\cdot\boldsymbol{x} = 0\) simply as a hyperplane and drop “affine”; because \(b=0\), the hyperplane goes through the origin of \(\mathbb{R}^n\).

For simplicity in this lecture, we will refer to an affine constraint as a linear constraint regardles off the value of \(b\).

A hyperplane splits the space it lies in into two halfspaces: the points that are “above” it and those that are “below” it. The next figure illustrates this. The halfspace “above” the hyperplane \(\boldsymbol{w}\cdot\boldsymbol{x} = b\) consists of points that satisfy \(\boldsymbol{w}\cdot\boldsymbol{x} \geq b\) and vice versa for the halfspace “below”.

Polyhedra as intersection of halfspaces#

When we have two or more linear inequality constraints, the intersection of the satisfying halfspaces forms the feasible region, a polyhedron. Polyhedra are a central structure in mathematical optimization. The next two figures illustrate the intersection of two halfspaces and a bounded polyhedron, also referred to as a polytope. Formally, a polyhedron is the set of points \(P\subset\mathbb{R}^n\) that satisfy a set of \(m\) linear inequalities of the form \(\boldsymbol{a}_j\cdot\boldsymbol{x}\leq b_j\) for \(j\in[m]\). The vectors \(\boldsymbol{a}_j\) are \(n\)-dimensional and \(b_j\) are scalars.

A polyhedron can be expressed compactly as

\[P=\{\boldsymbol{x}\in\mathbb{R}^n\;|\; A\boldsymbol{x}\leq\boldsymbol{b}\},\]

where we have stacked the \(\boldsymbol{a}_j\) vectors as rows in a matrix \(\boldsymbol{A}\in\mathbb{R}^{m\times n}\) and the \(b_j\) scalars as a single \(m\)-dimensional column vector \(\boldsymbol{b}\), i.e.,

\[\begin{split}\boldsymbol{A} = \left[ \begin{array}{ccc} {\rule[.5ex]{2.5ex}{0.5pt}} & \boldsymbol{a}^{\intercal}_{1} & {\rule[.5ex]{2.5ex}{0.5pt}} \\ {\rule[.5ex]{2.5ex}{0.5pt}} & \boldsymbol{a}^{\intercal}_{2} & {\rule[.5ex]{2.5ex}{0.5pt}} \\ & \vdots & \\ {\rule[.5ex]{2.5ex}{0.5pt}} & \boldsymbol{a}^{\intercal}_{m} & {\rule[.5ex]{2.5ex}{0.5pt}} \end{array} \right],\quad \boldsymbol{b} = \left[ \begin{array}{c} b_1\\ b_2\\ \vdots\\ b_m \end{array} \right].\end{split}\]

Linear programming#

Linear programming (LP) is central to optimization. Having defined polyhedra earlier, we will simply add the assumption that the objective function being minimized or maximized over a polyhedron is linear in the decision variables, i.e., for a parameter (user-provided) vector of objective function coefficients \(\boldsymbol{d}\in\mathbb{R}^{n}\), the objective writes

\[f(\boldsymbol{x}) = \boldsymbol{d}\cdot\boldsymbol{x} = \boldsymbol{d}^{\intercal}\boldsymbol{x} = \sum_{i=1}^{n}d_i x_i.\]

The next figure illustrates an LP on the polytope we have just looked at. Contour lines reflecting some objective function values are shown in greyscale. The objective function is optimized at corner points or vertices of the polytope. This will be the case in general: a linear objective function has a minimum or a maximum that is a vertex of the polytope representing the feasible region.

For a two-dimensional problem such as the one in the illustration, you can visually maximize the LP by “sliding the ruler” corresponding to the line that is orthogonal to the objective coefficient vector in the direction of that vector. To minimize the same objective, slide the line in the direction opposite to the objective vector. The last feasible point that the line touches is an optimum. Of course, such a method is not a general algorithm. In practice, LP solvers typically use either the Simplex method or an interior-point method. Simplex jumps from vertex to vertex, avoiding the interior of the polytope or any other points that are not vertices, until no neighboring vertex with a better value for \(f\) can be found. Interior-point methods start inside the polytope and find directions to move out towards its faces without exiting it.

Does the FONC still apply to LP?#

Recall that the first-order necessary condition (FONC) in unconstrained continuous optimization (see previous lecture) says that a local minimizer of a multivariate twice-differentiable function \(f\) satisfies \(\nabla_{\boldsymbol{x}}f(\boldsymbol{x}^\star) = \boldsymbol{0}\). Does it apply to LP?

Let’s look at the global minimizer in the last figure, call it \(\boldsymbol{x^\star}\); what is the gradient of the objective at \(\boldsymbol{x^\star}\)? Since \(f(\boldsymbol{x})=-3x_1-8x_2\), the gradient is \(\nabla_{\boldsymbol{x}}f(\boldsymbol{x}^\star)=[-3,-8]\) which is not equal to \([0,0]\) as required by the FONC for unconstrained optimization. This is despite \(\boldsymbol{x^\star}\) being a local (in fact global) minimum!

This is simply to show that more elaborate optimality conditions are required when an optimization problem is constrained.

Beyond linear programs#

In an LP, both the objective \(f(\cdot)\) and constraints \(c_j(\cdot)\) are affine functions of the decision variables \(\boldsymbol{x}\). A non-linear program (NLP) is one in which either the objective or the constraints are not linear. The next two figures have a non-linear objective function over linear constraints (a polytope) and a non-linear objective over a non-linear constraint (an ellipse, described as a quadratic function of \(\boldsymbol{x}\)), respectively.

Convexity#

Linear programs consist of a linear objective function and a polyhedral feasible region. The former is an example of a convex function whereas the latter is an example of a convex set.

Convex optimization deals with problems such as LPs but also more generally any optimization problem with a convex objective function and a convex feasible set. While we are yet to go over complexity theory, we can already note that convex optimization problems are “easy” to solve, i.e., they admit solution algorithms whose running times are polynomial in the “size” of the optimization problem (number of decision variables, number of constraints, etc.).

Definition 6 (Convex combination)

A convex combination of two vectors \(\boldsymbol{x}\) and \(\boldsymbol{y}\) is a vector \( \alpha\boldsymbol{x} + (1-\alpha)\boldsymbol{y} \) for some \(\alpha\in[0,1]\).

A convex combination of \(m\) vectors is a vector \(w_1\boldsymbol{v}^{(1)} + w_2\boldsymbol{v}^{(2)} + \dots + w_m\boldsymbol{v}^{(m)}\) with \(w_1,\dots,w_m\geq 0\) and \(\sum_{i=1}^{m}w_i=1\).

Definition 7 (Convex set)

A convex set is a set for which a line between any two points in the set is fully contained within the set. Formally, a set \(\mathcal{S}\) is convex if the following is true:

(3)#\[\begin{equation} \alpha\boldsymbol{x} + (1-\alpha)\boldsymbol{y} \in\mathcal{S} \end{equation}\]

for all \(\boldsymbol{x}, \boldsymbol{y}\in\mathcal{S}\) and for all \(\alpha\in[0,1]\).

Definition 8 (Convex function)

A convex function is a bowl-shaped function whose domain is a convex set. Any line connecting two points in the function’s domain does not lie below the function itself. Formally, a function \(f\) with domain a convex set \(\mathcal{S}\) is convex if the following is true:

(4)#\[\begin{equation} f(\alpha\boldsymbol{x} + (1-\alpha)\boldsymbol{y}) \leq \alpha f(\boldsymbol{x}) + (1-\alpha)f(\boldsymbol{y}) \end{equation}\]

for all \(\boldsymbol{x}, \boldsymbol{y}\in\mathcal{S}\) and for all \(\alpha\in[0,1]\).

SciPy is good for quick prototyping of LPs at small scale, perhaps up to hundreds of variables and constraints.

CVXPY is a widely used convex optimization solver with an easy-to-use programming interface; it cannot handle non-convex objectives and constraints.

Gurobi is a commercial integer programming solver with a free academic license for educational purposes. It is a highly reliable tool and can solve even non-convex QPs to global optimality!

IPOPT is an NLP solver. It can handle all four problem classes in the table above. It is typically very fast and guarantees locally optimal solutions for the most general class of problems, non-convex NLPs. Although it can handle LPs, Gurobi would be better in that case.
